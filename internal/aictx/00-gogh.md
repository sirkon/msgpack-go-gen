# Подробности работы с gogh

gogh - это библиотека для кодогенарации, закрывающая типичный бойлерплейт генерации кода на Go.

В данном случае вся инициализация была проведена изначально и поэтому работать нужно будет
только с конечным пунктом рендеринга - типом 

```go
type goRenderer = gogh.GoRenderer[*importer]
```

и (возможно) с функциями `gogh.Private` и `gogh.Public` для генерации "красивых" названий переменных -
они преобразуют camel case имена и последовательности строк в корректные соотв. pascalCase и PascalCase
имена, при этом оба имеют ввиду спецаббревиатуры, благодаря которым `gogh.Private("handler", "http")` 
превращается в `handlerHTTP`. 

## Импорты.

Вычисляют имена пакетов (`package name`). Автоматически определяют коллизиции и вводят алиасы. Позволяют
именовать импорты на уровне рендерера для последующего их использования в генерации. Работа с ними
выглядит так:

```go
r.Imports().Add("time").Ref("time")
r.Imports().Add("context").Ref("ctx")
r.Imports().Add("company/context").Ref("cmpctx")

r.L(`ctx, cancel := $ctx.WithTimeout($cmpctx.New(), $time.Second*5)`)
```

## Контекст рендерера.

**Вспомогательное**

У рендерера есть метод `r.Uniq("name", …)` который генерирует уникальное значение `nameXXX` в рамках спец-области. 
Если "name" внутри нет, то будет отдано "name". Если есть, то в зависимости от вызова `r.Uniq`
будет сгенерировано либо уникальное имя с добавленным индексом в суффиксе, либо в форме `r.Uniq("name", "opt")`
вначале попробует взять имя `"nameOpt"` и если оно тоже занято, то дальше уже пойдет вычисление суффикса для
уникализации.

Внимание: эта "спецобласть" НЕ СОВПАДАЕТ с ключами контекста. Это уникальная штука.

**Мутация контекста через Imports**
В предыдущем пункте ты увидел использование `Ref("xxx")`. Это есть одно из использований контекста:
мы добавляем имя пакета (или уже алиас имени пакета) под именем `xxx` в контекст рендеринга.

**Прямая мутация контекста**
Так же есть способ внести значение в контекст напрямую:

```go
r.Let("name", value)
```

**Встроенная мутация контекста**

```go
r.L(`@id := getIdentitifier()`)
```

что эквивалентно

```go
xxxid := r.Uniq("id")
r.Let("id", xxxid)
r.L("$id := getIdentifier()")
```

**Правила работы с выставлением записью значений в контекст**

Есть требование идемпотентности в некотором роде:

`r.Let("name", value)` паникует, если ключ `"name"` уже занят и его `value` отличается от нового.
Если `value` не отличается, то все норм.

Такое правило нужно для того чтобы нормально отрабатывали множественные `Imports.Add("context").Ref("ctx")`

### Scopes.

Рендерер может возвратить "дочерний" с помощью вызова `nr := r.Scope()`.
В рамках дочернего разрешено затемнение значений, т.е. мы можем переопределить там именованную
переменную на другое значение.
Мы можем получить родительский рендерер из дочернего с помощью `r.Parent()`. Но, внимание, если
наш рендерер уже самого высокого уровня, прямиком из `gogh.Package`, то вызов `Parent` вызовет панику.

## Рендеринг строк.

Осовными методами являются `L`, `S` и `N`. Отличие заключается в том, что `L` рендерит в буфер рендерера и
этот код окажется в результате. А `S` рендерит в строку. `r.N()` это эквивалент `r.L("")`

Так же можно использовать метод `Z()`, который возвращает "ленивый" рендерер:

> // Z provides a renderer instance for "laZy" writing.
> //
> // What it does:
> //  1. Inserts a new text block and switches the current
> //     renderer to it.
> //  2. Return a new renderer which references a block
> //     which was the current before.
> //
> // So, with this renderer you will write into the previous
> // "current", while the original renderer will write into
> // the next. This means you will have text rendered
> // with the returned GoRenderer instance will appear
> // before the one made with the original renderer after
> // the Z call. Even if writes with the original were made
> // before the writes with the returned.
> //
> // Example:
> //
> //	r.R(`// Hello`)
> //	x := r.Z()
> //	r.R(`// World!`)
> //	x.R(`// 你好`)
> //
> // Output:
> //
> //	// Hello
> //	// 你好
> //	// World!
> //
> // See, even though we wrote Chinese("Hello") after the
> // "World!" it appears before it after the rendering.

### Интерполяция в рендеринге.

**Позиционная**

Рендерер может принимать вариадик параметры при исполнении:

```go
r.L("$0 $1 $2", 1, "two", true) // "1 two true"
```

**Контекстная**

Рендерер может использовать именованные значения из скоупа:

```go
r.Imports().Add("context").Ref("ctx") // Т.е. Ref мутирует контекст текущего рендерера!
r.L(`ctx := $ctx.Background`)
```

### Поддерживаемые типы

В данном случае нам важно, что gogh умеет рендерить FQN для сущностей предоставляемых `go/types`.
Он берет на себя всю черновую работу по импорту (если он нужен в рамках пакета в котором будет находиться
генерируемый файл). Выглядит это так:

```go
var typ *types.Named
// ...
r.L`func (x *$0) Do() {`, typ) // func (x *pkgname.TypeName) Do() {
```
