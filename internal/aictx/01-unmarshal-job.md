# Генерация кода анмаршалера

Я подготовил заготовку функции осуществляющей генерацию кода unmarshaler-а, она лежит в файле

@generate_unmarshaler.go

и называется genUnmarshal.

Нужно добавить код анмаршалинга.

## Требования:

- Из-за того, что при анмаршалинге структур мы неминуемо наткнемся на `switch` по всем полям, я требую, чтобы
  анмаршалер полей представлялся в виде единственного вызова:

  ```go
  case "field_name":
      srcOffset, err = unmarshalFieldType(&dst.FieldName, src) // offset используется для вычисления следующего состояния src []byte
  ```
  или
  ```go
  case "field_name":
      srcOffset, dst.FieldName, err = unmarshalFieldType(data)
  ```
  первый вариант должен использоваться для контейнеров и структур, второй – для скалярных типов и указателей.
  Требование введено для снижения нагрузки на кеш инструкций.

- Для типов с генерируемым анмаршалом должен добавляться метод
  ```go
  func (x *XXX) UnmarshalMsgpack(src []byte) error
  ```
  где имя ресивера вычисляется аналогично тому, как это сделано в генераторе маршалера.
  Желательно ввести здесь кеш ресиверов и переиспользовать готовое имя ресивера.
- Нужно переиспользовать анмаршалеры для одного типа в рамках разных анмаршалеров.
- Если тип используется и как внутренний в другом типе, и как самостоятельный тип, то UnmarshalreMsgpack для этого
  типа должен вызывать сгенерированную свободную функцию.
- Публичные методы (MarshalMsgpack, UnmarshalPack) должны лежать ДО непубличных свободных функций. Используй
  Z()-метод рендерера, чтобы добиться такого поведения "бесплатно".
